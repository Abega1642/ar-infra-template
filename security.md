## Supported Versions

Only the latest version on the `main` branch is supported with security updates.

Older versions are not guaranteed to receive fixes.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

- **Do NOT** open a public GitHub issue.
- Contact the maintainer directly by email: <a.razafindratelo@gmail.com>

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact

You can expect an acknowledgment within **72 hours**.

## Disclosure Policy

After a vulnerability is reported, the maintainer will:

1. Validate and assess the issue
2. Prepare a fix
3. Release a patched version
4. Credit the reporter if desired
